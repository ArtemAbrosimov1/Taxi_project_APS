# Проект "Система диспетчеризации службы такси"

## Описание варианта №22

### Дисциплины системы:

**Источники:**
- ИБ - бесконечный источник заявок
- И32 - равномерное распределение

**Приборы:**
- П31 - экспоненциальное распределение времени обслуживания

**Дисциплины постановки в буфер (Д1):**
- Д1O3 - постановка в буфер если есть место
- Д1ОЗ2 - в порядке поступления

**Дисциплины отказа (Д1OO4):**
- Последняя поступившая заявка выбивается из буфера

**Дисциплины выбора заявок (Д2Б4):**
- Приоритет по номеру источника
- При равном приоритете - выбор самой новой заявки

**Отображение результатов работы программной модели:**
- ОР1 - сводная таблица результатов
- ОД1 - календарь событий, буфер и текущее состояние

### Бизнес-домен:

Система представляет собой службу такси с:
- **Клиентами** - создают заказы через приложение
- **Водителями** - обрабатывают заказы
- **Буфером** - очередь заказов на сервере
- **Диспетчером** - алгоритм распределения заказов

### Особенности реализации:

1. **Приоритеты заказов:**
   - HIGH ("Быстрый") - высокий приоритет
   - LOW ("Обычный") - низкий приоритет
  
   - Приоритеты ставятся рандомно для пошаговой реализации. В реальной ситуации клиенты определяют приоритет при заказе.

2. **Логика работы буфера:**
   - При переполнении выбивается последняя заявка
   - Заявки обрабатываются по приоритету источника
   - При равном приоритете выбирается самая новая заявка
  
   - Размер буфера для пошаговой реализации = 5. А количество водителей = 3

3. **Пошаговый режим:**
   - Отображение состояния системы после каждого действия
   - Календарь событий
   - Визуализация буфера и состояния водителей

### Запуск программы
При запуске необходимо выбрать действие:
1. Новый заказ от клиента ( генерируется заказ и ставится в буфер )
2. Обработать заказы из буфера ( заказы раздаются водителям и удаляются из буфера )
3. Освободить случайного водителя ( освобождается рандомный водитель )
4. Показать календарь событий
5. Выход

И в зависимости от выбранного пункта будет отображаться текущее состояние системы и 
производимые действия
